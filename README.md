# 1.旋转角度限幅；

基本要求：
由视觉反推出机器与墙面的夹角，初始角度定义为与幕墙90°垂直（相机正对着墙，相机发出的线与面前一个墙，两个组成的90度，既相机始终平行于墙），限制范围为正负45°（当相机不平行于墙的时候，计算相机的角度），
利用对极几何。
利用视觉里程计知识，原理为不同帧的图片之间存在一个角度差，需要了解简单视觉定位原理，利用图片反推出旋转和平动的位姿；
关键点在找到与墙面垂直的偏航角度，

# 2.幕墙清洁的高度


利用图像计算出需要清洁幕墙的高度
（假设现在在无人机上拍第一张照，无人机向上移动一米后拍第二张，两张照片都含有玻璃，我需要计算面前的墙的玻璃是多少米）（实际上这个假设很难保证，你也不知道无人机他上升过程到底是稳定上升，还是前后移动了，或者是一阵风吹偏了它），

需要简单的图片识别原理，
首先是玻璃边缘检测，其次根据深度/双目摄像头来算玻璃边缘的真实长度。
要求计算出的高度与实际的高度误差不超过10%



最简单的就是可以直接拍到一整个玻璃边框。 得到的玻璃画面最好是这样的→口。或者是井
但是假如玻璃较大，无法获取整个玻璃的图像，每次只能拍到玻璃的一部分。
比如只能看到左右的框和中间的一条线→|——|
或者是只能看到左右的框和摄像头刚好在玻璃中间，上面没有线，下面也没有线→|        |。
甚至极端情况下，左右玻璃的框也无法获得，每次都只有一条线一条线这样过去→——。或者是一条左右的线加上一条上下的线|——
最极端的情况下，你啥也没有，直接对着一个玻璃或者镂空的墙，不过这种可以当成背景之类。
这种情况下我推测必须借助无人机的移动情况推导。

实际上他甚至不是|——|，而是类似于/——\，或者是\——/。


以及，由于玻璃的透光性，镜面反射等，深度和红外可能会直接穿过玻璃或被打乱，有可能出错等。
所以目标是玻璃边缘的那些框架。

利用无人机搭载的视觉传感器，在贴近幕墙垂直上升的过程中，实时检测玻璃框架结构，计算每块玻璃面板的实际高度，并自动归类出大厦共有几种尺寸。

## Step 1:完全重置「以太网 3」+ 物理重连(5 分钟)
先做这一步,做完了把 `ipconfig` 输出贴我。
powershell

1) 释放当前租约
ipconfig /release "*以太网 3*"

2) 重置整张网卡的协议栈(关键:这一步会清掉之前所有 netsh 残留的静态 IP)
netsh int ip reset
netsh winsock reset

3) 在 PC 端操作完后,、、
    - 拔掉 TYPEC 线
    - 等 5 秒
    - 重新插入
    (让板子端 RNDIS 重新枚举)

4) 等待 **60 秒** (RDK X5 板子端 dnsmasq + gadget 启动可能慢)

5) 看 IP
ipconfig
成功标志: 「以太网 3」拿到了 192.168.128.x 的 IP(很可能是 192.168.128.10)。拿到后直接 `ping 192.168.128.1` 验证,通了就完事了。
失败标志: 还是 169.254.x.x → 进入 Step 2。

## Step 2:PowerShell cmdlet 配静态 + 关 IPv6(只在前一步失败时做)
powershell

1) 完全关掉 DHCP 客户端,不让它再抢
Set-NetIPInterface -InterfaceAlias "以太网 3" -DHCP Disabled

2) 用 PowerShell cmdlet 写静态 IP(.222 段不冲突的地址,不要用 netsh)
 不设网关,因为我们只走 L2 直连
New-NetIPAddress -InterfaceAlias "以太网 3" -IPAddress 192.168.128.222 -PrefixLength 24

3) 关 IPv6,免得它来回抢路由
Disable-NetAdapterBinding -InterfaceAlias "以太网 3" -ComponentID "ms_tcpip6"

 4) 验证 + ping 板子
ipconfig
ping 192.168.128.1

之后你可以让Ai帮你找一下wifi的地址，然后现在RDK里连接，而后可以借助vscode之类的，通过SSH连接，ssh root@192.168.0.111。
